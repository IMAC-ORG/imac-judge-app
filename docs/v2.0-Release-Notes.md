# AeroJudge Software v2.0 (2026 Ready)

Major updates for use with latest power board design, restructured figures/sequences, per-type round numbering, now supports mutliple Known/ Unknown sequences and Freestyle. Compatible with older devices (power board features unavailable).

---

## Folder Structure

- Flattened `figures/en/` structure for easier automated uploads [Issue 94]
- Created 2026 sequence structure in `figures/en/` folders
- Implemented `SequenceFolderResolver` with 5-case logic and fallback protection
- Updated audio folder structure
- Added fallback defaults: eyeball SVG for missing figures, audio fallback for missing figure numbers

---

## Round Numbering

- Sequential numbering by round type (e.g., Known 3, Unknown 1, Freestyle 1)
- Updated XML output to Score to reflect per-type numbering
- Round number used for `sequences.dat` lookup and folder resolution [Issue 70]
- Supports mutlple known/ unknown sequences in one competition
- Allows Score sequences.dat to control std vs alt sequence option
- **Incomplete KNOWN Round Warning:**
  - Detects when pilot completed Seq 1 but not Seq 2
  - Triggers when: 2-sequence competition AND pilot at sequence 2 AND KNOWN button clicked
  - Options: Continue Seq 2 | Start New Round | Exit for Review
  - API: `POST /api/pilot/{pilotId}/advance-round?type=KNOWN`

---

## Line Numbering

- Improved multi-line round numbering: Line 1 = rounds 1-20, Line 2 = 21-40, etc.
- Supports up to 8 flight lines without +10/+100 workarounds
- Note: Score currently doesn't parse line number from XML import (`XMLio.class`)

---

## Sequence Validation

- Added error checking for new/update contest and pilot sync [Issue 66]
- Validates sequence availability with graceful fallbacks (no white screen crashes)
- Fixed: pilot sync renamed and now properly refreshes sequence cache
- **Always-Show Validation Page:** Validation page now always displays after sync (eliminates timing issues from conditional redirects)
  - **Success state (green):** "All Sequences Validated!" with Continue button
  - **Warnings state (orange):** Warning list with Re-Sync + Continue buttons
  - **Errors state (red):** Error list with Re-Sync + Continue Anyway buttons

---

## Freestyle Support

- Added Freestyle as valid round type [Issue 103]
- Complete scoring functionality implemented and tested
- Output includes `duration_seconds` (default 240, editable in Score)

- **FREESTYLE Filter:** Added to pilot class filters (appears when any pilot has freestyle=true)

## Admin Pages

Redesigned admin interface with compact mobile-friendly layout. Header + nav buttons on single line for vertical space optimization. Accessible from error pages and pilot list settings.

- **`/admin/comp`** - Scorekeeper Admin
  - Known Sequence Count (Single/Double), Default Sequence Type (std/alt)
  - Refresh Current Event (update from Score)
  - Load NEW Event (archive current, load fresh) with confirmation modal
  - Local settings update via `POST /api/comp/local` (no Score contact required)

- **`/admin/device`** - Device Settings
  - Scoring Mode (Global / By Round / Defined Rounds)
  - Device Identity: Line Number, Judge Number (preloads current values)
  - Save & Load NEW triggers Score connection and event reload
  - **System Update:** Check for and install latest AeroJudge software updates
    - Primary source: GitHub (`IMAC-ORG/imac-judge-updater`)
    - Fallback: `aero-judge.com/update_mirror/`
    - API: `POST /api/system/update`

- **`/admin/scores`** - Score Admin
  - **Score Mismatch Resolution:** Detect and fix scores assigned to wrong pilots
    - Scans for mismatches between judge device and Score server
    - Resolution UI at `/admin/scores/resolve`
    - API: `GET /api/scores/mismatches`, `POST /api/scores/resolve`

- **`/newcomp`** - First Boot Wizard
  - 3-step setup: Event Settings → Device Identity → Load Event
  - Used only on fresh device setup

## Modal & Carousel Stability

- **Modal Cleanup on Navigation:** All open modals (except summary) are now properly closed when navigating carousel with prev/next buttons or numpad keys
- **Summary Modal Lock:** Carousel navigation is blocked while the score summary modal is open - judges must click "Review" or "Save" before continuing
- **Code Refactoring:** Navigation logic consolidated into `carouselPrev()` and `carouselNext()` functions for consistency
- **Applies to:** `judge.html`, `pilot-list-global.html`, `pilot-list-round.html`
- Fixes potential bugs from modals remaining in "open" state but hidden off-screen

---

## Error Handling

- **Global Error Page (`error.html`):** Catches unhandled 500 errors with user-friendly display
  - Red warning: "Please give this device to the Contest Director or Scorekeeper"
  - Admin button with confirmation modal (same warning as pilot list pages)
  - Prevents judges from being stuck on white screen errors

---

## Configurable Settings

- **Score Server Timeouts:** `score_poll_timeout` (default 2s) and `score_timeout` (default 10s) for Score communication
- **Local Settings API:** `POST /api/comp/local` updates sequences, sequenceType, and score_mode without Score server contact
